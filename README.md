# 價值迭代演算法視覺化工具 (Value Iteration Visualizer) 🤖

這是一個輕量化、互動式的網頁應用程式，使用 **Flask** 後端與 **原生 HTML/CSS/JS** 前端開發。旨在提供一個視覺化的直觀介面，幫助使用者理解強化學習 (Reinforcement Learning, RL) 中的核心概念——**價值迭代 (Value Iteration)** 演算法。

本專案允許您自定義一個 5x5 的網格世界 (Grid World)，設定起點、終點與障礙物，並觀察貝爾曼方程式 (Bellman Equation) 如何計算出每個狀態的最佳價值函數 $V(s)$ 以及最佳策略 $\pi(s)$。

---

## ✨ 核心特色

*   **🎨 互動式網格地圖：** 透過工具列輕鬆點擊網格，自定義起點 (S)、終點 (G) 以及障礙物 (Blocks)。
*   **⚡ 即時演算法運算：** 點擊執行後，後端將立即執行價值迭代，並將每一步的收斂結果回傳至前端。
*   **🔍 雙模式檢視切換：**
    *   **顯示價值 $V(s)$：** 查看每個格子在最佳狀態下所代表的預期未來獎勵總和。
    *   **顯示策略 $\pi(s)$：** 視覺化最佳路徑導航，以箭頭顯示每個狀態下應採取的最佳動作。
*   **🛡️ 安全選取模式 (Select)：** 內建預設的選取工具，防止在觀察結果時因誤觸網格而重置計算。
*   **📱 現代化設計：** 採用精美深色模式介面，並加入微互動動畫，提升使用者體驗。

## 🧠 演算法細節：價值迭代 (Value Iteration)

價值迭代是透過不斷重複執行 **貝爾曼最佳運算子 (Bellman Optimality Operator)** 來找到最佳價值函數的過程。

### 貝爾曼方程式 (Bellman Equation)
系統會對網格中的每一個狀態 $s$ 更新其價值：
$$V(s) \leftarrow \max_{a} \sum_{s', r} p(s', r | s, a) [r + \gamma V(s')]$$

在本專案的實作中：
1.  **狀態 (States)：** 5x5 網格中的座標 $(r, c)$。
2.  **動作 (Actions)：** 上、下、左、右四個方向。
3.  **折扣因子 ($\gamma$)：** 設定為 `0.9`，代表對未來獎勵的重視程度。
4.  **收斂門檻 ($\theta$)：** 設定為 `1e-4`，當所有狀態價值的最大變化量低於此數值時，演算法停止迭代。

## 🎮 環境規則與獎勵設定 (Reward Function)

為了引導演算法找到最短且安全的移動路徑，我們定義了以下獎勵規則：
*   **抵達終點 (Goal)：** $+10.0$ 分。
*   **撞擊障礙物 (Obstacle)：** $-100.0$ 分。
*   **一般移動 (Step Penalty)：** $-0.1$ 分（此設定是為了鼓勵演算法尋找步數最少的路徑）。
*   **邊界處理：** 若試圖走出網格邊界，會留在原位並扣除 $-1.0$ 分。

## 💬 深度解析與常見問題 (Q&A)

### Q1: 網格旁的「Select」工具作用是什麼？
**A:** 這是一個**安全檢視模式**。當跑完演算法後，任何對地圖的修改（如移動起點或增加障礙）都會導致原本的計算結果失效。啟用 `Select` 模式後，點擊網格不會觸發任何修改動作，讓您可以安心地觀察數值與箭頭，不用擔心誤觸導致畫面重置。

### Q2: 計算完成後，終點 (Goal) 自身的價值為什麼是「0」？
**A:** 這符合強化學習中**終止狀態 (Terminal State)** 的定義。終點是回合結束的地方，一旦到達就沒有「未來」可以評估。在數學實作中，我們將終點價值固定為 0，這能確保周圍格子計算出的價值完全來自於「踏入終點的那一瞬間所獲得的獎勵」，避免了價值在終點處自我循環產生的錯誤。

### Q3: 為什麼不直接把終點的價值設為 10？
**A:** 如果將終點 $V(G)$ 硬性設為 10，會導致**雙重計算 (Double Counting)** 錯誤。
在貝爾曼更新時，終點旁的格子會計算「踏入終點的獎勵 (10)」加上「折扣後的終點價值 ($10 \times 0.9 = 9$)」，結果變成 19。這會使得整個網格的數值被無止盡地放大，雖然箭頭方向可能正確，但數值本身將失去收斂的物理意義。
